## Применение SOLID

### 1. **Принцип единственной ответственности (SRP)**:
- Класс `VeterinaryClinic` отвечает только за проверку здоровья животных  
- Класс `ZooServices` управляет добавлением животных и расчётом еды

### 2. **Принцип открытости/закрытости (OCP)**:
- Новые виды животных (например, `Elephant`) добавляются через наследование от `Animal` без изменения существующего кода

### 3. **Принцип подстановки Лисков (LSP)**:
- Все животные реализуют `IAlive`, что гарантирует наличие свойства `Food`

### 4. **Принцип разделения интерфейсов (ISP)**:
- Интерфейсы `IAlive` (для живых существ) и `IInventory` (для инвентаря) разделены, чтобы избежать "раздувания" функциональности

### 5. **Принцип инверсии зависимостей (DIP)**:
- `ZooServices` зависит от абстракции `IVeterinaryClinic`, а не от конкретной реализации
